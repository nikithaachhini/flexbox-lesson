Flexbox

elegant layout mode for more civilized age

Different layouts

  • Block layout - vertically based layout used for laying out documents. Provides ability to float elements or to lay them over multiple columns.

  • Inline layout - horizontally based one used for laying out text.

  • Table layout - used for 2D tabular data.

  • Positioned layout - for explicit positioning.

Why Flexible box layout?

My goal in doing Flexbox and later Grid was to replace all the crazy float/table/inline-block/etc hacks that I’d had to master as a webdev
  Tab Atkins Jr.(primary author for those specs)
 

Issues

  • with float - need to do clearfix, no vertical centering.

  • with inline-block - "white-space" issue disturbs the layout

  • with table - complex code, responsive design, hacks

  • Earlier layouts can't support orientation changing, resizing and shrinking.

What is flex box?

  • It is used to layout, align and distribute space between elements in document even if their size is unknown.

  • It allows the container to alter its item's width/height as per the viewport/screen size.

  • It is direction agnostic so vertical centering, reordering, same height columns and changing direction is easy.

Order

  • flex items are displayed in same order as specified in source document.

  • This property controls how flex item is ordered in flex container.

  • ex: .flex-item { order: <integer>;}

Flex Item Properties

flex-grow

  • allows the flex item to grow if needed.

  • determines the amount of free space to be occupied by flex Item in flex Container.

  • Ex: .flex-item {flex-grow: <number>};

  • Default value is 0

Flex-grow Example 

Let's Chuck it

flex-basis

  • initial size of flex item before any space is distributed based on flex factors.

  • default value is auto - means initial width of flex item depends on size of its content.

  • Ex: .flex-item{flex-basis: <width> | <length>};

  • negative values are not allowed.

flex-basis with flex-grow

flex-shrink

  • allows the flex item to shrink if needed

  • means how much flex item would shrink with respect to other items when there is no enough space in flex container

  • Ex: .flex-item {flex-shrink: <number>};

  • default value is 1

Fixed Items

flex

  • short hand property for flex-grow, flex-shrink and flex-basis.

  • flex-shrink and flex-basis are optional.

  • default value is 0 1 auto.

  • Ex:  .flex-item{flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]};

flex Demonstration

align-self

  • used to align specific flex item.

  • possible to override the align-items property for specific flex items.

  • It accepts 5 values similar to align-items property.

  • Ex:  .flex-item {align-self: flex-start | flex-end | center | baseline | stretch};

So flexbox is amazing?

  • for small scale layouts - Yes! Defintely

  • flexbox is designed for component layouts in pages like navigation, forms, pagination etc.

  • for large scale layouts/complex two-dimensional ones  - Prefer "Grids"
